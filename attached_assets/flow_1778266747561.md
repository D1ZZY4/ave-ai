1. Sequence: Thinking Mode Expert

```mermaid
sequenceDiagram
    participant UI as Mobile UI (hooks)
    participant O as Orchestrator
    participant R as Rules Engine
    participant L as Ollama LLM (ctx 65536)
    participant T as Tool/Skill Executor
    participant S as Zustand Store

    UI->>O: dispatch({ type: "USER_INPUT", payload: {...} })
    O->>R: evaluateInput(input)
    R-->>O: OK
    O->>S: set(sessionActive: true, thinkingSteps: [])
    O->>L: POST /api/generate (prompt, system, tools schema)
    Note over L: Thinking #1 (stream)
    L-->>O: { thought: "...", tool_call: { name: "read_file" } }
    O->>S: appendThinkingStep({ step: 1, thought: "...", action: "read_file" })
    O->>R: preApproveTool("read_file", params)
    R-->>O: Approved
    O->>T: execute(read_file, params)

    R-->>O: OK
    O->>S: updateLastThinkingStep({ observation: "isi file..." })
    O->>L: lanjutkan dengan observasi
    Note over L: Thinking #2 dst.
    L-->>O: final_answer: "Ringkasan..."
    O->>R: evaluateOutput(final_answer)
    R-->>O: OK
    O->>S: setFinalAnswer, clearSession
    S-->>UI: Re-render dengan final answer + thinking steps
```

Thinking step direkam dengan tipe { step, thought, action, action_input, observation, timestamp }.

1. State Machine: Fast vs Expert

```mermaid
stateDiagram-v2
    [*] --> Idle
    Idle --> FastSession: Mode = Fast
    Idle --> ExpertSession: Mode = Expert

    state FastSession {
        [*] --> FastLoad: Load persona + rules
        FastLoad --> FastInputCheck: evaluateInput(input)
        FastInputCheck --> FastDenied: Input Ditolak
        FastDenied --> [*]
        FastInputCheck --> FastPrompt: Bangun prompt (tanpa alat)
        FastPrompt --> FastLLM: fetch Ollama (stream false)
        FastLLM --> FastOutputCheck: evaluateOutput(output)
        FastOutputCheck --> FastFilter: Filter/modifikasi
        FastFilter --> FastReturn: Return FinalAnswer
        FastOutputCheck --> FastReturn: OK
        FastReturn --> [*]
    }

    state ExpertSession {
        [*] --> ExpertLoad: Load persona + rules
        ExpertLoad --> ExpertInputCheck: evaluateInput(input)
        ExpertInputCheck --> ExpertDenied: Ditolak
        ExpertDenied --> [*]
        ExpertInputCheck --> ReActLoop: Mulai ReAct Loop
        state ReActLoop {
            [*] --> Think: LLM berpikir (stream)
            Think --> ActionSelect: Pilih tool/skill
            ActionSelect --> PreCheck: preApprove(tool, params)
            PreCheck --> Denied: Ditolak -> feedback
            Denied --> Think: Re-think
            PreCheck --> Execute: Disetujui
            Execute --> PostCheck: postCheckResult(result)
            PostCheck --> Blocked: Ditolak -> feedback
            Blocked --> Think: Re-think
            PostCheck --> Observe: OK
            Observe --> Think: Lanjut loop
            Observe --> FinalAnswer: Flag final
        }
        ReActLoop --> ExpertOutputCheck: evaluateOutput(final)
        ExpertOutputCheck --> ExpertFilter: Filter output
        ExpertFilter --> ExpertReturn: Return FinalAnswer
        ExpertOutputCheck --> ExpertReturn: OK
        ExpertReturn --> [*]
    }
```

State diimplementasikan dengan useReducer atau XState, transisi dijaga type guard.
